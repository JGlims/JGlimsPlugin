KEY TECHNICAL DETAILS
Custom Model Data Format (Paper 26.1.2 / 1.21.4+)
Items use minecraft:select in items/*.json:

Copy{
  "model": {
    "type": "minecraft:select",
    "property": "minecraft:custom_model_data",
    "index": 0,
    "fallback": {"type": "minecraft:model", "model": "minecraft:item/diamond_sword"},
    "cases": [
      {"when": "excalibur", "model": {"type": "minecraft:model", "model": "minecraft:item/excalibur"}}
    ]
  }
}
"index": 0 is REQUIRED or Paper silently fails
All files must be UTF-8 without BOM
Legendary weapons use setStrings(List.of("weapon_id")) on CustomModelDataComponent
Vampire/werewolf/drops use deprecated setCustomModelData(int) which sets floats[0]

BetterModel Resource Pack Generation
BetterModel auto-generates a resource pack at plugins/BetterModel/pack/BetterModel-Pack.zip on startup/reload. This contains texture atlases, model JSONs, and overlays. These MUST be merged into our main resource pack via build_rp.py. The command /bettermodel reload or bm reload via rcon forces regeneration.

Elytra Glide Animation Trick (for vampire bat flight)
To show elytra gliding animation without actual elytra:

Call player.setGliding(true)
Listen to EntityToggleGlideEvent and cancel it when the player should be gliding (prevents server from resetting the state)
This shows the smooth horizontal glide pose instead of running animation
Item 3D Depth
"parent": "minecraft:item/handheld" = 3D held-out look (for swords/weapons)
"parent": "minecraft:item/generated" = flat paper look (for materials/consumables)
Weapons should use handheld, non-weapons should use generated
